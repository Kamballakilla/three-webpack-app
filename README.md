## Оптимизация

### Оптимизация объектов

Для рендера кубов используется InstancedMesh, что позволяет отрисовывать тысячи объектов одним draw call вместо создания 1000 отдельных Mesh.
Трансформации каждого экземпляра задаются через setMatrixAt, обеспечивая эффективное обновление позиций, вращения и масштаба без лишней нагрузки на CPU.
Для label реализовано плавное затухание (distance-based fading) — прозрачность автоматически уменьшается с увеличением расстояния до камеры, снижая визуальный шум и повышая читаемость сцены.

### Обработка кликов при оптимизации сцены

Для уменьшения draw calls объекты объединяются в один `merged mesh`.

На фронте выполняется raycast по объединенной геометрии.  
Данные пересечения отправляются на сервер через WebSocket.

На сервере хранятся `BoundingBox` и геометрия объектов до объединения.  
По координатам пересечения сервер определяет исходный объект и при необходимости возвращает геометрию отдельного mesh.

Такой подход позволяет:
- рендерить сцену как один mesh;
- сохранять точную обработку кликов;
- динамически восстанавливать отдельные объекты из объединенной геометрии.

### Структура хранения данных 

Для эффективной работы с данными используется несколько индексов хранения:

```js
{
  items: Array,
  byId: Map,
  byCoord: Map
}

items — основной массив всех объектов. Используется для итерации, рендера и последовательной обработки данных.
byId — индекс для быстрого поиска объекта по id.
byCoord — индекс для быстрого поиска объекта по координатам.

Использование Map позволяет получать объект за O(1) вместо линейного поиска через Array.find() (O(n)).
```

# Grid Layout

Этот алгоритм выполняет простое размещение объектов в сцене по принципу построчной упаковки без пересечений.

Как работает алгоритм:
Объекты размещаются последовательно слева направо по оси X
Когда следующий объект не помещается в текущую строку (превышает SCENE_WIDTH), происходит перенос на новую строку
Новая строка начинается ниже предыдущей по оси Z
По оси Y все объекты фиксированы (0)
Каждый объект позиционируется по центру (учитывается width / 2 и depth / 2)
